# Introduction

The `@antelopejs/interface-file-storage` package defines a provider-agnostic file storage interface for AntelopeJS applications. It provides a consistent API for uploading, downloading, and managing files through presigned URLs, regardless of the underlying storage backend (Amazon S3, Google Cloud Storage, Azure Blob Storage, etc.).

## Core Concepts

The interface centers on presigned URLs for file transfers. Instead of routing file data through your application server, clients upload and download directly to and from the storage provider using time-limited, signed URLs.

| Function | Purpose |
| --- | --- |
| `CreateUploadUrl` | Generate a presigned URL for direct file uploads |
| `CreateReadUrl` | Generate a URL to read or download a file |
| `DeleteFile` | Remove a file from storage |
| `FileExists` | Check whether a file exists |
| `GetFileMetadata` | Retrieve metadata for a stored file |
| `PromoteFile` | Move a staged upload out of the staging area to its final key |
| `MoveFile` | Move a stored object from one resource key to another |

## Installation

```bash
npm install @antelopejs/interface-file-storage
```

## Visibility

The `Visibility` type controls file access. It accepts two values:

| Value | Description |
| --- | --- |
| `"public"` | Files are accessible via a direct, permanent URL with no expiration |
| `"private"` | Files require a presigned URL with a limited expiration window |

Visibility is determined by the storage configuration rather than per-request parameters. When calling `CreateReadUrl`, the response reflects the file's visibility: public files return a URL with no `expiresAt`, while private files include an expiration timestamp.

```ts
import { CreateReadUrl } from "@antelopejs/interface-file-storage";

// For a public file, the URL has no expiration
const publicFile = await CreateReadUrl("uploads/logo.png");
console.log(publicFile.url);       // Direct public URL
console.log(publicFile.expiresAt); // undefined

// For a private file, specify an expiration time in seconds
const privateFile = await CreateReadUrl("documents/contract.pdf", 3600);
console.log(privateFile.url);       // Presigned URL
console.log(privateFile.expiresAt); // Timestamp when the URL expires
```

## Error Classes

The interface defines two error classes for common failure scenarios.

### `UploadValidationError`

Thrown when an upload request violates the specified constraints. The `code` property indicates the specific validation failure:

| Code | Description |
| --- | --- |
| `"SIZE_EXCEEDED"` | The file size exceeds the maximum allowed size |
| `"MIMETYPE_NOT_ALLOWED"` | The file's MIME type is not in the allowed list |

```ts
import { CreateUploadUrl, UploadValidationError } from "@antelopejs/interface-file-storage";

try {
  await CreateUploadUrl(
    { filename: "video.mp4", size: 500_000_000, mimetype: "video/mp4" },
    { maxSize: 10_000_000 },
  );
} catch (error) {
  if (error instanceof UploadValidationError) {
    console.error(`Validation failed [${error.code}]: ${error.message}`);
  }
}
```

### `FileNotFoundError`

Thrown when an operation targets a file that does not exist in storage. The error message includes the resource key that was not found.

```ts
import { GetFileMetadata, FileNotFoundError } from "@antelopejs/interface-file-storage";

try {
  const metadata = await GetFileMetadata("uploads/missing-file.txt");
} catch (error) {
  if (error instanceof FileNotFoundError) {
    console.error(error.message); // "File not found: uploads/missing-file.txt"
  }
}
```

## Multi-Bucket Support

Every function accepts an optional `storage` parameter as its last argument. This identifier selects a specific storage bucket in multi-bucket configurations. When omitted, the default storage is used.

```ts
import { CreateUploadUrl } from "@antelopejs/interface-file-storage";

// Upload to the default storage
await CreateUploadUrl({ filename: "photo.jpg", size: 204800, mimetype: "image/jpeg" });

// Upload to a specific storage bucket
await CreateUploadUrl(
  { filename: "photo.jpg", size: 204800, mimetype: "image/jpeg" },
  undefined,
  "media-bucket",
);
```
