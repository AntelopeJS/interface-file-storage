# File Operations

The interface provides functions for uploading files via presigned URLs, generating download URLs, deleting files, checking existence, and retrieving metadata.

## Upload Flow

Uploading a file is a two-step process:

1. Your server calls `CreateUploadUrl` to generate a presigned upload URL.
2. The client sends the file directly to the storage provider using that URL.

This pattern keeps file data off your application server and reduces bandwidth costs.

### The `UploadRequest` Object

The `CreateUploadUrl` function requires an `UploadRequest` describing the file to upload:

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `filename` | `string` | Yes | Original filename |
| `size` | `number` | Yes | Exact file size in bytes |
| `mimetype` | `string` | Yes | MIME type of the file (e.g., `"image/jpeg"`) |
| `path` | `string` | No | Path or folder prefix for organizing the file |
| `metadata` | `Record<string, string>` | No | Custom key-value metadata to attach to the file |

### The `UploadConstraints` Object

The optional `UploadConstraints` parameter validates the request before generating a URL. If validation fails, an `UploadValidationError` is thrown.

| Field | Type | Description |
| --- | --- | --- |
| `maxSize` | `number` | Maximum allowed file size in bytes |
| `allowedMimetypes` | `string[]` | List of permitted MIME types |

### The `PresignedUploadResponse` Object

A successful call returns a `PresignedUploadResponse`:

| Field | Type | Description |
| --- | --- | --- |
| `uploadUrl` | `string` | Presigned PUT URL for direct upload |
| `resourceKey` | `string` | Unique key to reference this file in future operations |
| `expiresAt` | `number` | Timestamp when the presigned URL expires |
| `headers` | `Record<string, string>` | Required headers the client must include with the PUT request |

### Complete upload example

The following example demonstrates the full server-side flow for generating an upload URL with validation:

```ts
import { CreateUploadUrl, UploadValidationError } from "@antelopejs/interface-file-storage";

async function handleUploadRequest(file: { name: string; size: number; type: string }) {
  try {
    const response = await CreateUploadUrl(
      {
        filename: file.name,
        size: file.size,
        mimetype: file.type,
        path: "user-uploads/avatars",
        metadata: { uploadedBy: "usr_abc123" },
      },
      {
        maxSize: 5 * 1024 * 1024, // 5 MB
        allowedMimetypes: ["image/jpeg", "image/png", "image/webp"],
      },
    );

    // Return the presigned URL and headers to the client
    return {
      uploadUrl: response.uploadUrl,
      resourceKey: response.resourceKey,
      headers: response.headers,
      expiresAt: response.expiresAt,
    };
  } catch (error) {
    if (error instanceof UploadValidationError) {
      throw new Error(`Upload rejected: ${error.message}`);
    }
    throw error;
  }
}
```

The client then performs the upload using the presigned URL:

```ts
// Client-side upload using the response from the server
async function uploadFile(file: File, presignedData: { uploadUrl: string; headers: Record<string, string> }) {
  const response = await fetch(presignedData.uploadUrl, {
    method: "PUT",
    headers: presignedData.headers,
    body: file,
  });

  if (!response.ok) {
    throw new Error("Upload failed");
  }
}
```

> **Important:** The client must include all headers from the `headers` field in the PUT request. The presigned URL signature covers `Content-Type` and `Content-Length`, so any modification to these values causes the request to be rejected.

## Download Flow

The `CreateReadUrl` function generates a URL to access a stored file.

### The `PresignedReadResponse` Object

| Field | Type | Description |
| --- | --- | --- |
| `url` | `string` | URL to access the file |
| `expiresAt` | `number \| undefined` | Expiration timestamp (`undefined` for public files) |

### Generate a download URL

```ts
import { CreateReadUrl } from "@antelopejs/interface-file-storage";

// Generate a read URL with 1-hour expiration (for private files)
const { url, expiresAt } = await CreateReadUrl("user-uploads/avatars/photo.jpg", 3600);

console.log(url);       // Presigned or public URL
console.log(expiresAt); // Expiration timestamp, or undefined if public
```

### Serve files in an API response

```ts
import { CreateReadUrl } from "@antelopejs/interface-file-storage";

async function getFileDownloadUrl(resourceKey: string) {
  const { url, expiresAt } = await CreateReadUrl(resourceKey, 300); // 5 minutes

  return {
    downloadUrl: url,
    expiresAt,
  };
}
```

## Delete a File

The `DeleteFile` function removes a file from storage.

```ts
import { DeleteFile } from "@antelopejs/interface-file-storage";

await DeleteFile("user-uploads/avatars/old-photo.jpg");
```

To delete a file from a specific storage bucket:

```ts
import { DeleteFile } from "@antelopejs/interface-file-storage";

await DeleteFile("backups/archive-2025.zip", "backup-bucket");
```

## Check File Existence

The `FileExists` function returns `true` if the file exists, or `false` otherwise.

```ts
import { FileExists } from "@antelopejs/interface-file-storage";

const exists = await FileExists("user-uploads/avatars/photo.jpg");

if (!exists) {
  console.log("File not found, using default avatar.");
}
```

## File Metadata

The `GetFileMetadata` function retrieves detailed information about a stored file. It throws a `FileNotFoundError` if the file does not exist.

### The `FileMetadata` Object

| Field | Type | Description |
| --- | --- | --- |
| `resourceKey` | `string` | Unique resource key |
| `filename` | `string` | Original filename |
| `size` | `number` | File size in bytes |
| `mimetype` | `string` | MIME type |
| `lastModified` | `number` | Last modified timestamp |
| `metadata` | `Record<string, string>` | Custom metadata attached to the file |

### Retrieve file metadata

```ts
import { GetFileMetadata, FileNotFoundError } from "@antelopejs/interface-file-storage";

try {
  const meta = await GetFileMetadata("user-uploads/avatars/photo.jpg");

  console.log(meta.filename);     // "photo.jpg"
  console.log(meta.size);         // 204800
  console.log(meta.mimetype);     // "image/jpeg"
  console.log(meta.lastModified); // 1711800000000
  console.log(meta.metadata);     // { uploadedBy: "usr_abc123" }
} catch (error) {
  if (error instanceof FileNotFoundError) {
    console.error("The requested file does not exist.");
  }
}
```

## Complete Workflow Example

The following example ties together the full lifecycle of a file: upload, read, inspect metadata, and delete.

```ts
import {
  CreateUploadUrl,
  CreateReadUrl,
  GetFileMetadata,
  DeleteFile,
  FileExists,
} from "@antelopejs/interface-file-storage";

// 1. Generate a presigned upload URL
const upload = await CreateUploadUrl({
  filename: "report.pdf",
  size: 1_048_576,
  mimetype: "application/pdf",
  path: "reports/2026",
});

console.log("Upload to:", upload.uploadUrl);
console.log("Resource key:", upload.resourceKey);

// 2. After the client uploads, generate a read URL
const read = await CreateReadUrl(upload.resourceKey, 3600);
console.log("Download from:", read.url);

// 3. Retrieve metadata
const metadata = await GetFileMetadata(upload.resourceKey);
console.log("File size:", metadata.size, "bytes");

// 4. Check existence before deleting
const exists = await FileExists(upload.resourceKey);
if (exists) {
  await DeleteFile(upload.resourceKey);
  console.log("File deleted.");
}
```
